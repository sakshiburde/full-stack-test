## 2. How would you track down a performance issue in production? Have you ever had to do this?

When investigating a performance issue in production, I follow a systematic process:

1. Gather information from users, logs, and monitoring tools.
2. Identify whether the issue originates from the frontend, backend, database, network, or server.
3. Check server resources such as CPU, memory, disk usage, and network activity.
4. Analyze application logs and error reports.
5. Profile database queries to identify slow or inefficient operations.
6. Use browser developer tools to inspect network requests, rendering performance, and JavaScript execution.
7. Reproduce the issue in a staging environment when possible.
8. Apply optimizations and verify improvements through testing and monitoring.

In my professional experience, I have worked on React-based applications where performance issues were caused by unnecessary component re-renders, large API responses, and inefficient data fetching. I resolved these issues by optimizing React component rendering, reducing API calls, implementing lazy loading, and improving overall application responsiveness.
